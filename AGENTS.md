# 易经衍化之旅

本文件由 `/home/gezhengewen/yijing-derivation/CLAUDE.md` 迁移而来，作为 Codex 项目规则。

## 概述

易经完整推演链路的交互式单页可视化应用。
纯 HTML5 + CSS3 + JavaScript，零依赖。

## 内容

13 个推演阶段 + 多个专题页面（波形比对、博弈论、基因组、算法动画等）。

## 数据量

64 卦卦辞 + 384 条爻辞 + 12 条道德经 + 12 个骨架卦。

## 协作风格

直接给结果，不总结。

## 前提审计规则

任何内容型输出（论文、叙事、可视化、文案）交付前，必须附一段「前提审计」，否则不算完成。

审计清单：

1. 前提审查：这段内容依赖了哪些假设？有证据吗？把假设显式列出。
2. 术语审查：用的术语是“学界惯例”还是“经过验证的事实”？惯例不等于正确。
3. 因果审查：在描述相关性还是在暗示因果？因果需要完整证据链。
4. 反面审查：存在哪些合理的反对意见？必须提及，不能跳过。

输出格式：在内容末尾附加一个「前提审计」区块，逐条列出。如果所有条目都无问题，写“已审查，无未验证假设”。

历史教训：“伏羲序→文王序”这个框架曾被使用但未被质疑。数学正确不代表历史叙事正确；邵雍在文王之后约 2000 年才重构“伏羲序”，文王不可能“重排”一个不存在的东西。生成内容时必须主动挑战前提。
