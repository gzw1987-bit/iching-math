## 1. 背景与定位

项目研究六十四卦二进制自然序与传世文王序之间的置换，并以论文、验证器和跨学科交互页面公开展示。

**[事实]** 固定当前六位编码与两份序列后，置换循环型为 (52,10,2)，零固定点，置换阶为 260。  
**[事实]** 项目已经通过 GitHub Pages 公开，并有 DOI 10.5281/zenodo.19143997。  
**[推断]** 正确定位是“一个可复现的有限置换观察 + 有边界的历史讨论 + 明确标识的探索性类比”，而不是对文王设计意图、历史因果或跨学科同构的证明。

## 2. 用户与核心场景

- 数学读者：独立复算循环、汉明距离与随机基线。
- 易学/历史研究者：核对序列来源、编码、年代与文献。
- 普通读者：用交互页面理解有限置换。
- 作者/审校者：在发布前检查计算、来源、叙事边界与版本一致性。

## 3. 问题与价值

- 数学正确可能被错误转译为历史因果或设计意图。
- 编码、方向和索引约定变化会改变置换。
- 事后选择“罕见结构”存在多重比较/观察者偏差。
- 跨领域共享数字或图形不等于严格同构。
- 本地修订、GitHub Pages 和 DOI 版本可能不一致。

价值在于提供透明数据、独立代码、可反驳结论和清晰认识论边界，让数学观察经得起复算而不被叙事放大。

## 4. 目标与非目标

### 目标

- 核心数据、编码、方向、算法和统计检验完全可复现。
- 数学事实、统计描述、历史主张、语义解释和艺术类比分级发布。
- 历史与新颖性主张有可审计检索/引用记录。
- 所有发布载体使用同一指标定义和版本。
- 每份内容交付包含前提审计。

### 非目标

- 不证明文王重排了后世邵雍二进制序。
- 不用罕见循环型证明非随机、神秘来源或设计意图。
- 不把卦象与基因、经济周期、博弈、信息论的相似性称作严格同构，除非定义保结构映射。
- 不把探索页中的叙事作为历史事实。

## 5. 当前实现与审计基线

### 已验证

- **[事实]** verify_core 全部通过：64 唯一编码、(52,10,2)、0 固定点、阶 260、汉明分布、均值 3.349、偶:奇步 48:15。
- **[事实]** 2,000,000 次 Monte Carlo 得到精确循环型频率 0.0959%，理论值 1/1040=0.0962%。
- **[事实]** verify_site 检查 21 个文档通过。
- **[事实]** GitHub Pages 当前 status=built、HTTPS 开启；公开页面 HTTP 200。
- **[事实]** DOI 会重定向至 Zenodo 记录。

### 关键缺口

- **[事实]** verify.py 因缺少 cast_hexagram/yaoci_data 无法运行；genome_analysis 同样依赖仓库外模块。
- **[事实]** ruff 报告 21 个问题。
- **[事实]** README、论文和公开首页将随机相邻汉明距离基线写为 3.0；随机排列不放回的精确期望应为 6×32/63=3.047619。
- **[事实]** 50 万次独立模拟中，均值至少达到 3.349 的单侧频率约 2.37%；该结果仍需预先定义检验与多重比较说明。
- **[事实]** 偶:奇汉明步 48:15 对应奇步仅 15；在 32/32 奇偶标签随机排列下，奇步≤15 的精确概率约 1.04×10⁻⁵，但不能反推历史意图。
- **[事实]** 公开首页仍把 3.2:1 标成“cycle positions”，实际是相邻文王卦的汉明距离奇偶性。
- **[事实]** 本地 7 月修订尚未提交，GitHub Pages 基于 6 月版本，公开页缺少部分最新边界修正。
- **[事实]** 中文论文结论仍写“不是随机的”；genome、timeline、wave 等页面残留“证明、必然、因果链、同构”等越界措辞。
- **[事实]** 新颖性声明列出检索平台和关键词，但没有完整检索日期、结果快照、纳排标准和可复核 bibliography。

## 6. 功能需求与验收标准

| 编号 | 需求 | 验收标准 |
| --- | --- | --- |
| FR-01 | 权威输入 | 两序列、卦名和编码来源逐条引用；文件有哈希与版本。 |
| FR-02 | 编码契约 | 明确爻读取方向、0/1、索引和映射方向；替代约定作为敏感性分析。 |
| FR-03 | 核心验证 | 至少两套独立实现产生相同循环、阶和汉明结果。 |
| FR-04 | 随机基线 | 使用不放回精确期望；报告统计量、零假设、试验数、seed、置信区间和多重比较。 |
| FR-05 | 历史来源 | 年代、作者归属和“习称伏羲序”采用学术来源，并区分传统归属与现代研究。 |
| FR-06 | 新颖性审计 | 保存检索式、平台、日期、结果、筛选规则；只表述“在所检索范围未发现”。 |
| FR-07 | 内容分级 | 每页标为 verified computation、historical note、speculative analogy 或 artistic exploration。 |
| FR-08 | 类比门禁 | “同构”必须给集合、运算、双射和保持证明；否则统一称形式类比。 |
| FR-09 | 出版一致性 | 本地、GitHub Pages、论文 HTML/PDF/DOI 的版本、指标和边界一致。 |
| FR-10 | 自包含复现 | 公开列出的脚本不依赖仓库外未说明模块；一条命令运行全部门禁。 |
| FR-11 | 可访问性 | 所有链接、交互、双语标签和移动端通过浏览器测试。 |
| FR-12 | 前提审计 | 每次内容变更附假设、术语、因果和反面审查。 |

## 7. 非功能需求

- **数学正确性：** 精确结果有断言，统计结果有误差与 seed。
- **史学严谨性：** 传统归属、年代与现代重构分开表述。
- **可复现性：** 零隐藏依赖，版本化数据和输出。
- **出版诚信：** DOI 版本不可与在线页面静默漂移。
- **可维护性：** 静态检查、链接、内容 guardrail 和浏览器测试自动化。
- **无障碍：** 关键结论不只靠颜色/动画表达。

## 前提审计

1. **前提审查：** 本 PRD 以当前 HEXAGRAM_MAP、六位自下而上编码和文王序为固定输入；更换编码或映射方向会改变具体置换。核心复算有代码证据，历史年代与作者归属仍依赖文献审校。
2. **术语审查：** “文王序”“伏羲序”为传统/学术惯称，不等于历史作者事实；“显著”“同构”“非随机”只有满足明确统计或代数定义时才能使用。
3. **因果审查：** 循环、汉明和 parity 只描述两个固定排序的关系，不能推出文王设计意图、文化因果或跨学科共同来源。
4. **反面审查：** 合理反对意见包括编码选择效应、后验选指标、多重比较、文本/年代争议以及小有限状态空间自然产生相似结构。
