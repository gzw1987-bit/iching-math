## Page and Interaction Issues

1. README lists three missing pages.

   `README.md` links to `engine-architecture.html`, `strategy-debate.html`, and `deep-diagnostic.html`, but those files are absent from this checkout. The homepage does not link them, so this is mainly a README/release integrity issue.

2. `verify.py` and `genome_analysis.py` are listed as project contents but are not self-contained.

   Both import `cast_hexagram.py`; `verify.py` also imports `yaoci_data.py`. Those files are not in this repo. `py_compile` passes because imports are not resolved at compile time, but normal execution will fail unless users know to provide external skill files on `PYTHONPATH`.

3. README and homepage use different guardrail strength.

   `index.html` uses a more careful phrase: "We have not found this cycle type discussed in the literature we surveyed." `README.md` says the cycle structure "has not been previously reported in the mathematical or sinological literature." The README phrasing is stronger than the documented evidence.

4. Homepage label for the parity metric is ambiguous.

   `index.html` labels `3.2:1` as "even:odd ratio" with subtitle "parity of cycle positions." The recomputed value is the parity of adjacent Hamming distances in King Wen order (`48:15`), not cycle-position parity. This should be relabeled to avoid a false metric.

5. No repo-local premise/audit rule document is present.

   The paper pages contain methodological warnings, but there is no root-level `CLAUDE.md` or explicit audit checklist to enforce them across exploratory pages. This makes drift likely as new visualizations are added.

## Mathematical Narrative Risks

1. Core permutation result is reproducible, but adjacent metrics need precise wording.

   The `[52, 10, 2]`, zero fixed points, and order `260` claims check out using the README's permutation array. The `3.349` Hamming mean also checks out, but only under the page's specific definition: adjacent hexagrams by King Wen number `1 -> 64`, using the six-bit encoding in `HEXAGRAM_MAP`. It should not be described as a property of the README array's adjacent entries, which gives a different value.

2. "Not random" is overstated in several exploratory pages.

   The paper correctly says the cycle type is uncommon but not statistically decisive and is not evidence of intentional design. Other pages drift beyond this boundary:
   - `genome-view.html` says `[52,10,2]` proves the structure is "not random" and "purposeful/directed."
   - `coupled-cycles.html` frames cycle independence as evidence of King Wen's design intent.
   - `derivation-journey.html` uses broad "isomorphism" language across genetics, communications, E8, octonions, and particle physics.

   These are publication risks unless clearly marked as speculative analogy, not mathematical conclusion.

3. The "genome rearrangement" analogy is too categorical.

   `genome-view.html` uses terms like "genomics verdict," "96.8%," "far beyond 300 million years," and "directed chromosome recombination event." As a visualization metaphor, this may be useful; as written, it reads like a cross-domain scientific inference. The project should either downgrade the page to an explicitly speculative analogy or remove it from the publication path.

4. The 10-cycle semantic reading is handled well in `paper-en.html` but not consistently elsewhere.

   `paper-en.html` warns that cycle semantics are conjectural and high-risk for pattern perception. `coupled-cycles.html` presents the same interpretive layer as a system architecture and design-intent argument. This violates the paper's own boundary: cycle structure is fact; semantic interpretation is conjecture.

5. One random baseline appears inconsistent.

   `derivation-journey.html` includes "随机期望~4.2个循环" in an isomorphism panel, while the paper and verifier state the expected number of cycles is approximately `H_64 = 4.74`. Use `4.74` consistently, or remove the inline random expectation from secondary panels.

## Guardrail Compliance

Implemented well:

- `paper-en.html` includes a historical clarification that King Wen did not rearrange the later binary order.
- `paper-en.html` explicitly warns that semantic readings are conjecture.
- `paper-en.html` states that `0.094%` is not evidence of intentional design.
- `index.html` marks the Kondratieff page as speculative.

Not implemented consistently:

- README does not include the paper's strongest caveats around novelty, statistical significance, and semantic interpretation.
- Exploratory cross-disciplinary pages often present analogies as conclusions.
- The homepage labels only one speculative page, while genome/game/coupled-system/coupled-cycles/four-minds style pages also need speculation labels or route separation.

## Publish-Blocking Fixes

1. Fix README integrity.

   Remove missing links or add the missing files:
   - `engine-architecture.html`
   - `strategy-debate.html`
   - `deep-diagnostic.html`

2. Make Python reproduction self-contained or documented.

   Either include `cast_hexagram.py` and `yaoci_data.py`, or mark `verify.py` and `genome_analysis.py` as requiring external data modules with a short setup section. For publication, a core `verify_cycle.py` with no external imports would be preferable.

3. Harmonize the novelty claim.

   Replace README's "has not been previously reported" with the safer homepage/paper version: "we have not found this cycle type in the publicly searchable literature surveyed." Include the surveyed sources and search date if this remains a publication claim.

4. Relabel the `3.2:1` metric.

   Suggested homepage label: "even:odd Hamming steps" and subtitle: "adjacent King Wen pairs." Avoid "permutation ratio" and "cycle positions."

5. Add a publication boundary banner or badges.

   Pages should be categorized as:
   - Verified computation
   - Historical/methodological note
   - Speculative analogy
   - Artistic/interactive exploration

   The current homepage only marks Kondratieff as speculative, but several other pages need the same treatment.

## Recommended Content Downgrades Before Public Release

- `genome-view.html`: rewrite as "genome rearrangement analogy" and remove "proves not random," "purposeful," "directed," and biological-time claims unless sourced and justified.
- `coupled-cycles.html`: replace "design intent evidence" with "one possible interpretive reading; not implied by the cycle decomposition."
- `derivation-journey.html`: downgrade "同构本质" claims to "形式类比" unless a precise structure-preserving map is defined and cited.
- `README.md`: add a short "Methodological boundaries" section matching the paper:
  - binary natural order postdates King Wen;
  - cycle decomposition is mathematical fact;
  - semantic readings are conjectural;
  - exact cycle type rarity is not evidence of intentional design;
  - novelty claim is limited to surveyed public literature.
